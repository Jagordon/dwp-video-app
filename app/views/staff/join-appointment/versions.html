Remote consultation - staff facing prototype

This is for testing only, do not rely on it if you want something real to happen.

Latest version

Version 2

Assumption: Users will be able to determine from a list of participants how many are present in the appointment.

Hypotheses

Because:

  • there are security concerns around uninvited attendees being able to enter the appointment
  • we have observed attendees trying to access their appointment from multiple devices
We believe that:
  • displaying a list of participants in the call to the host
Will result in:
  • a higher level of security
  • greater control of the appointment
  • fewer issues with video feeds
We'll know this is true:
  • when observations show that work coaches are able to identify when participant(s) are in the call
  • when feedback shows the work coach has investigated any unexpected participants and taken the appropriate action.

Version details
  • On both journeys, 'Waiting room' heading added to participant list.
  • On both journeys, the host can 'admit' an attendee to the appointment once they have joined the 'Waiting room'. The attendee then moves to under the 'Participants' heading.
  • On both journeys, permissions page removed.
  • For the link journey, there is no call to action on the final page.
  • For the link journey, the start page is skipped to reduce the number of steps in to the appointment.
Assumptions

Link

  • A link will be easier to access the appointment than a code.
  • Users will have greater control over the start of the appointment and over who can join and when.
  • Users will close the window after the appointment has ended.

Code

  • Using a code to access the appointment will mean users have more control over the different windows on their desktop.
  • Users who have multiple appointments in a day, will keep the 'Conduct a video appointment' service open.
  • Users will have greater control over the start of the appointment and over who can join and when.
Research findings
  • Work coaches spoke of the benefits of checking their environment and ensuring they were ready to start the appointment before admitting the attendee.
  • Majority noticed the change from nobody in the waiting room to an attendee and intuitively admitted. There were some comments about the 'admit' link being too small.
  • The link was viewed as quicker and simpler than going back and forth to copy the code.
  • Everyone clicked straight in to the appointment, even with the Microsoft Edge pop up.
  • At the end of the appointment, most work coaches understood they could just close the screen on the link journey but some thought they may also be closing UCFS.
  • On the code journey, work coaches understood they could leave that Edge window open all day and join appointments. A couple confused 'Start a new appointment' for creating a new one.

Testing version 1 - link on DOI journey

Assumptions
  • One device will improve the experience of both host and attendee.
  • A link to the specific appointment (rather than code entry) will be a simpler journey for the host.
  • A responsive video screen will make it simple to split the screen and perform the tasks required whilst on the video call.
  • At the end of the appointment, they will want to close the video service to return to other tasks.
  • Picture-in-picture functionality will give the host flexibility to perform the tasks required whilst on the video call.
Research findings
  • All workcoaches found it easier to use one device compared to two.
  • No problems with the code were identified, and it could now be pasted across.
  • The majority of work coaches thought that a link would be quicker to use than the code.
  • Splitting the screen wasn't instinctive but once prompted, most split the screen vertically.
  • All tasks to be done within an appointment could be when using the video service.
  • A number of work coaches also used Word or OneNote whilst completing the appointment.
  • Majority of work coaches found the picture-in-picture version the most user friendly.
  • All workcoaches were able to navigate back to the video service from picture-in-picture but in different ways.
  • Not all work coaches were comfortable in identifying the Edge browser.
