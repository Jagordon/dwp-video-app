Assumptions

Link

  • A link will be easier to access the appointment than a code.
  • Users will have greater control over the start of the appointment and over who can join and when.
  • Users will close the window after the appointment has ended.

Code

  • Using a code to access the appointment will mean users have more control over the different windows on their desktop.
  • Users who have multiple appointments in a day, will keep the 'Conduct a video appointment' service open.
  • Users will have greater control over the start of the appointment and over who can join and when.
Research findings

Testing version 1 - link on DOI journey

Assumptions
  • One device will improve the experience of both host and attendee.
  • A link to the specific appointment (rather than code entry) will be a simpler journey for the host.
  • A responsive video screen will make it simple to split the screen and perform the tasks required whilst on the video call.
  • At the end of the appointment, they will want to close the video service to return to other tasks.
  • Picture-in-picture functionality will give the host flexibility to perform the tasks required whilst on the video call.
Research findings
  • All workcoaches found it easier to use one device compared to two.
  • No problems with the code were identified, and it could now be pasted across.
  • The majority of work coaches thought that a link would be quicker to use than the code.
  • Splitting the screen wasn't instinctive but once prompted, most split the screen vertically.
  • All tasks to be done within an appointment could be when using the video service.
  • A number of work coaches also used Word or OneNote whilst completing the appointment.
  • Majority of work coaches found the picture-in-picture version the most user friendly.
  • All workcoaches were able to navigate back to the video service from picture-in-picture but in different ways.
  • Not all work coaches were comfortable in identifying the Edge browser.
