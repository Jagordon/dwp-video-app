Replace background controls

Proposed link journey - November 2021

Hypotheses

Because:

  • we know when some users join the meeting first, they use the refresh button multiple times before the other person joins
  • we know some users do not like seeing the large view of themselves as it makes them uncomfortable
We believe that:
  • by explaining they have joined the appointment, but the other person hasn't
  • by making the large view of themselves a smaller 'selfie' view
Will result in:
  • users understanding what is happening and that they do not need to do anything
  • users feeling more comfortable
We'll know this is true:
  • when users give us feedback that they understand what is happening
  • when users give us feedback that they feel more comfortable with a smaller 'selfie' view
  • if there is a reduction in the use of the refresh button prior to the second person joining

Because:

  • we know users want privacy from the start of the appointment
  • we know hosts are not always the first in to the appointment
We believe that:
  • letting users preview their video before joining the appointment
  • letting users hide their background before joining the appointment
Will result in:
  • further privacy
  • a smoother start to the appointment
  • work coaches feeling more comfortable joining the appointment
We'll know this is true:
  • when users give us feedback that they would like to set their background before entering the appointment

Interstitial start page alternative - April 2021
Interstitial start page with preview - April 2021

Hypotheses
  1. Small selfie: If we show a smaller view of themselves than the other participants, work coaches will feel at ease and be able to focus on the other participants in the appointment.
  2. Equal selfie: If we show all feeds as equal size in a grid format, work coaches will be more engaged in the conversation and know they can be seen clearly.
  3. Fill frame: If we show a cropped, zoomed in view of other participants, work coaches will be able to read facial expressions and build rapport.
  4. Fit-to-frame: If we show the whole frame of other participants, work coaches will be able to see everything they need to see.
  5. Blur background: If we give the option to blur the background, this will provide sufficient privacy for the work coach.
  6. Cover background: If we give the option to fully replace the background with a pre-selected image, this will provide privacy and professionalism for the work coach.
  7. Radio button option: If we provide radio buttons to change their background setting once in the appointment, the work coach will understand whether hide background is on or off, and how to change it.
  8. Link in header option: If we provide a link in the header to change their background setting once in the appointment, the work coach will quickly understand the action and change if required.
  9. Settings drop down: If we provide a settings menu with background change, as we assume it will only be required once per appointment, we will have a simple interface with less distractions.
  10. Interstitial start page: If we provide an interstitial start page with the option to change their settings and screen layout before entering the appointment, the work coach will have a more professional and prepared start to the appointment, adding value to the existing start page.
  11. Link in to the appointment: If we provide a link rather than a code to join the appointment, the work coach will have a more time efficient and simpler journey.
  12. End page: If we provide clear instructions on the end page, the work coach will close the page and return to other tasks once the appointment is finished.


Testing version 2a - waiting room (code)

Testing version 2b - waiting room (link- opens in new window)

Version details
  • On both journeys, 'Waiting room' heading added to participant list.
  • On both journeys, the host can 'admit' an attendee to the appointment once they have joined the 'Waiting room'. The attendee then moves to under the 'Participants' heading.
  • On both journeys, permissions page removed.
  • For the link journey, there is no call to action on the final page.
  • For the link journey, the start page is skipped to reduce the number of steps in to the appointment.
User needs
  • As a workcoach, I need to know when the attendee is ready to start the appointment, so that I can start the appointment when I am ready.
  • As a workcoach, I need to formally start the appointment at a time of my choosing with a citizen who is ready, so that I can manage my time effectively.
Assumptions

Link

  • A link will be easier to access the appointment than a code.
  • Users will have greater control over the start of the appointment and over who can join and when.
  • Users will close the window after the appointment has ended.

Code

  • Using a code to access the appointment will mean users have more control over the different windows on their desktop.
  • Users who have multiple appointments in a day, will keep the 'Conduct a video appointment' service open.
  • Users will have greater control over the start of the appointment and over who can join and when.
Research findings
  • Work coaches spoke of the benefits of checking their environment and ensuring they were ready to start the appointment before admitting the attendee.
  • Majority noticed the change from nobody in the waiting room to an attendee and intuitively admitted. There were some comments about the 'admit' link being too small.
  • The link was viewed as quicker and simpler than going back and forth to copy the code.
  • Everyone clicked straight in to the appointment, even with the Microsoft Edge pop up.
  • At the end of the appointment, most work coaches understood they could just close the screen on the link journey but some thought they may also be closing UCFS.
  • On the code journey, work coaches understood they could leave that Edge window open all day and join appointments. A couple confused 'Start a new appointment' for creating a new one.

Testing version 1 - link on DOI journey

User need
  • As a workcoach, I need to access other applications without losing the view of the video, so that I can do other tasks during the appointment.
Assumptions
  • One device will improve the experience of both host and attendee.
  • A link to the specific appointment (rather than code entry) will be a simpler journey for the host.
  • A responsive video screen will make it simple to split the screen and perform the tasks required whilst on the video call.
  • At the end of the appointment, they will want to close the video service to return to other tasks.
  • Picture-in-picture functionality will give the host flexibility to perform the tasks required whilst on the video call.
Research findings
  • All workcoaches found it easier to use one device compared to two.
  • No problems with the code were identified, and it could now be pasted across.
  • The majority of work coaches thought that a link would be quicker to use than the code.
  • Splitting the screen wasn't instinctive but once prompted, most split the screen vertically.
  • All tasks to be done within an appointment could be when using the video service.
  • A number of work coaches also used Word or OneNote whilst completing the appointment.
  • Majority of work coaches found the picture-in-picture version the most user friendly.
  • All workcoaches were able to navigate back to the video service from picture-in-picture but in different ways.
  • Not all work coaches were comfortable in identifying the Edge browser.
