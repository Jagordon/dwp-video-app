Version 2

Hypotheses

Because:

  • within the UC service, the user is given individual boxes for the required information to be copied in to
  • the help link is a static link and can be saved elsewhere
  • we have observed confusion around where the help link should be copied to once separated out
We believe that:
  • removing the content around the links
  • removing the help link from this page and giving it in a more appropriate way
Will result in:
  • a smoother and quicker flow between our create service and the use case
We'll know this is true:
  • when observations show users copying the correct information into the correct place

Version details: 2.0, 2.1

2.0

  • Removal of the help link from the video confirmation page.
  • Finish button content updated to 'Create another appointment'.
  • 'Create another appointment' now takes the user to the 'Enter appointment details' page.
  • 'Close' button added if create service opened in new tab.
  • Link removed from inset text, as it has the same function as the main call to action.
  • Content updated to be use case agnostic on video confirmation page.
  • Long URL wraps around within container on smaller screen sizes.

2.1

  • Removed 'Cancel' from the 'Enter appointment details'page as there is no current need for it.
Issues this addresses 2.0

2.0

  • Confusion over where to put the help link. This is no longer required and is not individual to each appointment.
  • We assume the close button will encourage some tabs to be closed once done, returning them to the next part of the journey.
  • Finish button content updated to be more representative of its function. Going straight to the 'Enter appointment details' page reduces the number of clicks.
